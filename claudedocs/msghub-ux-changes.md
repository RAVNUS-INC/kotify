## 1. 사용자 관점 Before/After

### 발송 경험

```
[Before — NCP]
메시지 입력 → 42 byte (SMS) → 발송 → 5초 폴링 → 결과

[After — msghub]  
메시지 입력 → 42 byte · 단문 · RCS 8원 (3명 = 24원) → 발송 → 웹훅 즉시 → 결과 + 실제 비용
```

**사용자가 추가로 할 일: 없음.** 동일하게 메시지 쓰고 보내기만 하면 됨.

### 결과 확인

```
[Before]
번호 | 상태 | 결과 | 결과코드 | 완료시각
(비용 정보 없음)

[After]
번호 | 채널 | 상태 | 결과 | Fallback 사유 | 완료시각
+ 캠페인 요약: 총 비용 936원 (RCS 72건 576원 + SMS 25건 225원 + LMS 5건 135원) | 실패 3건 제외
```

## 2. 발송 페이지 (`/compose`)

### 2.1 바이트 카운터 + 요금 표시

**Before:**
```
42 byte (SMS)
```

**After:**
```
42 byte · 단문 → RCS 양방향 8원/건 (미지원 시 SMS 9원)
```

```
156 byte · 장문 → RCS LMS 27원/건 (미지원 시 LMS 27원)
```

```
이미지 첨부 · 이미지 → RCS 이미지 40원/건 (미지원 시 MMS 85원)
```

### 2.2 미리보기 영역 (HTMX `/compose/preview`)

**Before:**
```
┌─ 미리보기 ─────────────────────┐
│ 유형: SMS                      │
```

```
┌─ 미리보기 ─────────────────────────────────────┐
│ 유형: 단문                                      │
│ 발송 채널: RCS 양방향 → 미지원 시 SMS 자동 전환 │
│ 유효 수신자: 100명                              │
│ 잘못된 번호: 2건                                │
│                                                 │
│ 예상 비용                                       │
│   전체 RCS 성공 시: 800원 (8원 × 100명)         │
│   전체 SMS 전환 시: 900원 (9원 × 100명)         │
│   예상 범위: 800 ~ 900원                        │
└─────────────────────────────────────────────────┘
```

**이미지 첨부 시:**

## 3. 대시보드 (`/`)

### 3.1 통계 카드: 4개 → 5개

**Before (4카드):**
```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│오늘 발송 │ │이번달    │ │이번달    │ │ 진행 중  │
│ 캠페인   │ │ 캠페인   │ │ 발송성공 │ │          │
│   12     │ │   45     │ │  1,234   │ │    2     │
└──────────┘ └──────────┘ └──────────┘ └──────────┘
```

**After (5카드):**
```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│오늘 발송 │ │이번달    │ │이번달    │ │ 진행 중  │ │ RCS     │
│ 캠페인   │ │ 캠페인   │ │ 발송성공 │ │          │ │ 도달률   │
│   12     │ │   45     │ │  1,234   │ │    2     │ │  72%    │
└──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘
```

**RCS 도달률 카드:**
- 이번 달 전체 발송 중 RCS로 최종 전달된 비율
- 계산: `(RCS 채널 성공 수 / 전체 성공 수) × 100`
- 클릭 시 → `/campaigns` (전체 이력)

### 3.2 최근 발송 이력 테이블 뱃지

**Before 유형 뱃지:**
```
SMS (green) | LMS (neutral) | MMS (orange)
```

**After 유형 뱃지:**
```
단문 (green) | 장문 (neutral) | 이미지 (orange)
```

**채널 뱃지 (신규 열 또는 유형 뱃지 옆):**
- 완료된 캠페인: `RCS 72% · SMS 28%` 형태 간략 표시
- 진행 중: 기존과 동일

### 3.3 체크리스트 변경

**Before:**
```
✓ 발신번호 등록
✓ 연락처 등록
```

**After:**
```
✓ 발신번호 등록
✓ 연락처 등록
✓ RCS 브랜드 연동
```

## 5. 캠페인 상세 (`/campaigns/{id}`)

**가장 많이 바뀌는 페이지.**

### 5.1 기본 정보 카드

**Before:**
```
┌─ 기본 정보 ──────────────────────────┐
│ 발신번호: 02-1234-5678   유형: SMS   │
│ 발송 시각: 04-17 14:30   상태: 완료  │
└──────────────────────────────────────┘
```

**After:**
```
┌─ 기본 정보 ─────────────────────────────────────┐
│ 발신번호: 02-1234-5678    유형: 단문              │
│ 발송 시각: 04-17 14:30    상태: 완료              │
│                                                   │
│ 발송 비용: 820원                                  │
│   RCS 72건 × 8원 = 576원                         │
│   SMS 25건 × 9원 = 225원                         │
│   LMS  2건 × 27원 = 54원  ← LMS fallback 발생 시 │
│   실패 3건 = 과금 제외 (-24원)                    │
└───────────────────────────────────────────────────┘
```

### 5.2 채널 분포 바 (개선된 디자인)

진행률 바와 통합된 2단 구조:

```
┌─ 발송 결과 ─────────────────────────────────────┐
│                                                   │
│  완료 97 / 100명                                  │
│  ┌──────────────────────────────────────────────┐ │
│  │████████████████████████████│██████████│░░░░░░│ │
│  │       RCS 72 (72%)        │ SMS 25   │실패 3│ │
│  └──────────────────────────────────────────────┘ │
│                                                   │
│  ✓ 성공 97  ×  실패 3  ⏱ 대기 0                  │
│                                                   │
│  채널 상세:                                       │
│    RCS  ████████████████████  72명 (74%)  576원   │
│    SMS  ██████                25명 (26%)  225원   │
│                                                   │
│  총 비용: 801원 (실패 3건 제외)                    │
│                                                   │
└───────────────────────────────────────────────────┘
```

**디자인 요소:**
- 메인 바: 3색 스택 (RCS=blue #3b82f6, SMS/LMS/MMS=green #16a34a, 실패=red #dc2626)
- 채널 상세: 각 채널별 미니 바 + 건수 + 비용
- 총 비용: bold, 실패 건 제외 명시

### 5.3 수신자 결과 테이블

**Before 컬럼:**
```
번호 (130px) | 원본 입력 (130px) | 상태 (80px) | 결과 | 결과코드 (70px) | 완료시각 (130px)
```

**After 컬럼:**
```
번호 (130px) | 채널 (70px) | 상태 (70px) | 결과 | Fallback 사유 (160px) | 비용 (60px) | 완료시각 (120px)
```

| 열 | 변경 | 설명 |
|----|------|------|
| 번호 | 유지 | 수신번호 (mono) |
| **채널** | **신규** | `RCS` (blue), `SMS` (green), `LMS` (neutral), `MMS` (orange) 뱃지 |
| 상태 | 유지 | 성공/실패/대기 뱃지 |
| 결과 | 유지 | 결과 설명 (코드 통합) |
| **Fallback 사유** | **신규** | RCS 실패 시 사유 (예: "단말 미지원") |
| **비용** | **신규** | 건당 비용 right-aligned (예: "8원", "9원", "-") |
| 완료시각 | 유지 | mono |
| 원본 입력 | **제거** | 공간 확보 |
| 결과 코드 | **제거** | 결과 열에 통합 |

**예시 행:**
```
01012345678 │ RCS  │ 성공 │ 발송 완료     │              │  8원 │ 14:30:15
01087654321 │ SMS  │ 성공 │ 발송 완료     │ 단말 미지원   │  9원 │ 14:30:18
01055556666 │ LMS  │ 성공 │ 발송 완료     │ 타임아웃      │ 27원 │ 14:30:20
01099998888 │  —   │ 실패 │ 비가입자/결번 │ 단말 미지원   │  —  │ 14:30:22
```

### 5.4 상태 필터 탭

**Before:**
```
[전체 100] [성공 97] [실패 3] [대기 0]
```

**After:**
```
[전체 100] [성공 97] [실패 3] [대기 0]  │  채널: [전체] [RCS 72] [SMS 25] [LMS 2]
```

- 기존 상태 필터 유지


### 5.6 비용 계산 로직

```python
PRICE_TABLE = {

    "RCS_SMS": 17,       # RCS SMS (사용 안 함, 양방향 대신)
    "RCS_LMS": 27,       # RCS LMS

    "LMS": 27,

# 리포트의 productCode + ch로 비용 결정
# 실패 (resultCode != "10000") → 비용 0
# 캠페인 총 비용 = sum(성공 건별 비용)
```

## 8. 발신번호 관리 (`/admin/callers`)

### 8.1 테이블 변경

**Before:**
```
번호 | 표시이름 | 상태 | 기본 | 등록일 | 관리
```

**After:**
```
번호 | 표시이름 | 상태 | RCS | 기본 | 등록일 | 관리
```

**RCS 열:**
- `✓ 양방향` (blue badge) — RCS 챗봇 등록된 번호
- `—` (text-muted) — 미등록

### 8.2 발송 시 RCS 등록 번호만 허용

발신번호 선택 드롭다운에서:
- RCS 등록 번호: 정상 표시
- RCS 미등록 번호: `(RCS 미등록)` 라벨 추가 + 선택 불가 (disabled)

또는 더 간단하게:
- **RCS 미등록 번호는 발신번호 목록에서 비활성 처리**
- 관리자 힌트: "RCS 양방향으로 발송하려면 msghub 콘솔에서 챗봇으로 등록된 번호만 사용 가능합니다."

### 8.3 Caller 모델에 RCS 상태 필드 추가

```python
class Caller:
    # 기존 필드 유지
    rcs_enabled: bool = False    # RCS 챗봇 등록 여부 (신규)
    rcs_chatbot_id: str | None   # 챗봇 ID (신규, nullable)
```

### 8.4 힌트 텍스트

**Before:**
```
숫자 외 문자는 자동 제거됩니다. NCP에 사전 등록된 번호만 정상 발송됩니다.
```

**After:**
```
숫자 외 문자는 자동 제거됩니다. msghub 콘솔에서 사전 등록된 번호만 사용 가능합니다.
RCS 발송을 위해 해당 번호가 챗봇(양방향)으로 등록되어 있어야 합니다.
```

### 8.5 RCS 상태 토글 (관리자)

관리자가 발신번호별로 "RCS 등록됨" 체크를 수동 설정:

```
번호 추가 폼:
  전화번호: [          ]
  표시이름: [          ]
  RCS 등록: [✓]         ← 신규 체크박스
  [추가]
```
